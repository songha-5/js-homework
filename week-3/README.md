# 과제 수행 과정 기록
HTML/CSS작업을 하루동안 작업하였습니다. 스크린리더를 고려하여 작성하고자 하였는데 role이나 다른것들을 찾아보고 이해하는 시간이 더 오래 걸렸던거 같습니다. 그리고 지금도 사실 잘 적용됬는지 걱정이됩니다.

JS는 차례대로 어떤것을 구현해야할지, 어떤것이 바뀌어야하는지 어떤 반복문을 돌려야할지 정리하고 코드를짰습니다.

for문을 사용해서 탭 버튼 이벤트를 추가하였습니다. 그 중 오래걸렸던 문제가 하나 있었습니다. 탭이 active되었을때 다른탭을 누르면 active된 탭에 class를지우고 해당 탭에 active class를 추가하는것이였는데요. 

처음에는 조건문을 걸지않고 버튼을 누르면 무조건 remove를 실행하게 하였습니다. 그런데 버튼을 누르다보니 같은버튼을 여러번 누르면 불필요한 remove가 중복실행되는걸 늦게 알아버렸고 꼼수로 active있는거만 선택해서 삭제하게 하였습니다.

계좌조회, 투자현황, 최근 거래 목록, 카드관리를 배열로 관리하고 반복문을 통해 데이터를 넣는 방식을 적용하고 싶었습니다. 그런데 배열안의 숫자나 ',' '-' '+' '.'등을 따로 분리해서 관리하고싶었는데 분리가 안됬고, 마이너스 금액이나 퍼센트같은 경우 class="minus"를 추가하고 싶었는데 배열에서 해당값을 뺄 수 없어서 class도 추가하지 못했습니다.

# 과제 코드 리뷰

- 웹 접근성을 올리고자 하였습니다.
  - aria-selected, role, aria-labelledby, aria-controls을 추가하여 지난주보다 웹 접근성을 개선하고자 하였습니다.
  - h1을 추가하여 어떤 코드인지 추가 설명하였고, sr_only를 추가하였습니다.

- 탭버튼을 눌렀을때 중복코드가 실행되지 않도록 방지하였습니다.
  - remove를 무조건 실행되게 짰었는데 조건문을 걸어서 해당 클래스를 가지고있는거만 지우도록 하였습니다.

- 배열과 반복문을 통해 카드리스트를 관리하고자 하였습니다.
  - 전체 리스트를 HTML이 아닌 배열로 관리해서 배열만 추가하면 카드리스트가 늘어나거나 줄어들도록 하였습니다.
  - HTML의 기존 텍스트 코드들을 남겨놨는데 지워야하는지 고민입니다.
  - +5.2, 123-456-789012같은 데이터들의 관리를 좀 더 생각해봐야 할거 같습니다.
  - "-"값들에게 class가 추가되어야하는데 전혀 추가되지않아 디자이너가 원하는 스타일이 나오지 않았습니다.

# 회고 (깨달은 점, 아쉬운 점 등)

조건문을 상황에 맞게 사용할 수 없었습니다. 처음 tab버튼을 반복문을 돌릴려고 for of를 사용하고자 하였는데, 결국 왜 안되는지 이해못한 상태로 for문을 돌려서 완성했습니다. forEach는 index를 반환하고, for문도 index를 반환하니까 가능한거같은데 for of은 왜 안되는지, 어디서 잘 적용해야하는지 잘 모른다는걸 알았습니다. 반복문은 공부가 더 필요한거 같습니다.

아직 배열을 어디서 어떻게 써야하는지 잘 모르겠습니다. 반복문을 돌려 받아오는 값들을 넣어주려면 배열을쓰는게 맞아보였습니다. 그런데 그 안의 숫자값들이나 +, -값들을 구분해서 class를 추가하고싶은데 그러려면 값을 따로 빼서 관리해야하나 싶어서 고민이 됩니다. 그리고 값을 따로 빼서 관리하려고해도 어떻게 해야할지 몰라 헤매이게 됩니다

그리고 만약 받아오는 값들이 정해져있다면 어디서부터 어떻게 오는지몰라서 제가 어떻게 가공해서 넣어야하는지 감이 잡히지 않습니다. 다양하게 연습해볼 기회인데 제가 그걸 잘 활용하지 못하는거 같아 아쉽습니다.

스크린리더에서 어떻게 읽힐지 어느순서가 읽기 편한지 고민이 많이 됩니다. 

카드숫자에 -나 공백을 넣어서 관리하고싶은데 찾아보니 시간내에 이해하고 적용할 수 없어서 아쉬웠습니다.
